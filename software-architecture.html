Software System Architecture

Why Cloud and Why AWS?

In consideration with the agileness, scalability of system, ease of deployment and management and cost efficiency, a natively cloud-based software system architecture is proposed to back the monitoring system.

Cloud architecture allows incremental development and deployment by taking various resources and services as building blocks, making the software engineering project agile. Well-designed interfaces from and to the resources allow components to be plugged into the system easily, resulting in high scalability. Cloud resources and resources are charged on pay-as-you-go basis, which larges reduces the cost from that for an on-premises architecture.

As an example, Amazon Web Services (AWS) is chosen to illustrate how such cloud architecture can be built, other cloud services providers (e.g. Microsoft Azure or Google Cloud) would also serve the purpose well. The following diagram envisions the cloud architecture in a high-level overview.

Technological Stacks

Viewed in a bottom-up approach, the architecture is comprised of the following stacks: Hardware Integration, Database, Machine Learning Suite, Serverless  API Suite, Content Distribution, Logs and Assets, Clients and Management.

Hardware Integration

Deployed hardware devices in the lagoon will regularly send collected data to a message queue (Amazon SQS), in which the requests will be redirected to the suitable processing cloud functions (AWS Lambda) via an IoT centre (AWS IoT Core). The results will be stored in a NoSQL database.

Database

A NoSQL database (Amazon DynamoDB) is selected for scalability, compatibility with RESTful APIs and ease of updating the schemas. GraphQL can be integrated easily to resolve future complexity when the system expands.

Machine Learning Suite

A machine learning engine (TensorFlow on AWS) is deployed to regularly train models based on the data from the database to predict trends or discover underlying patterns and structures, providing data-driven insights that could lead to more intelligent solutions to improve the ecosystem in and around the lagoon.

Serverless API Suite

In coming requests from frontend will be queued and redirected to the suitable processing functions via an API processor (Amazon API Gateway), which protects the functions behind it via authentications and authorisations. Security headers, API keys and CORS policies can be managed via such API processors as well for easy API management and deployment.

Content Distribution

A global content distribution network (CDN) will be deployed (Amazon CloudFront) to redirect HTTPS requests from the users to the required resources. For instance, <domain.name.com>/ requests will be redirected to static asset origins for so that the webpage assets can be served; <domain.name.com>/api/* requests will be redirected to the Serverless APIs to utilise requested services. Authorisation and authentication functions are also attached to the requests so that the services are protected. Caching policies will be configured at the CDN to speed up the delivery globally, regardless of which regions the resources are deployed in.

Logs and Assets

Storage services (Amazon S3) are deployed to host static assets and system logs.

Clients

A BI dashboard (Amazon QuickSight) and frontend clients (websites, mobile apps or desktop clients) will be provided to supply understandable data visualisation. Machine learning capabilities can be utilised and configured on these clients as well.

Management

System logs and metrics (Amazon CloudWatch) are visualised and presented to system managers for system monitoring and warnings. A cost and usage monitor (AWS Cost & Usage Report) provides the operation managers with the cost and usage statics to empower project budgeting.

Cost Estimation

It is estimated that such a system built on AWS will cost 280.93 USD per month, the cost will vary based on actual usage on pay-as-you-go basis.
